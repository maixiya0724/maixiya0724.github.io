TypeScript 教程

人生不值得

之前一直都想要找时间把ts的只是给巩固一下 ，找个时间抽离一下

1. 安装

npm install -g typescript

安装完成以后在node 命令行里面 通过 tsc dome.ts 命令 可以把 ts编译为js

  1. 开发工具如何自动编译 ts文件
  • vscode
    安装完成以后 在ts终端里面运行 tsc –init 生成typescript.config.js 的配置文件 然后修改里面的输出目录 outDir 指定输入的js目录来进行
  1. 在终端里面 运行任务 选择监视ts 然后可以边修改ts文件边编译成js 是不是很方便呢

2. TS 的数据类型

ts中定义类型的时候会增加类型校验 在声明的时候必须指定类型

string 类型
var a:string =  'rerer'
number 类型 
var b:number = 24343
boolean 类型
var c:boolean = true
Array 类型 
第一种
var d:number[] = [1,2,3] // 定义里面的元数据都是number类型
第二种
var e:Array<number>= [11,22]
  • 枚举类型

  • 任意类型

    let a:any = 14

    a =’43434’ // 不会报错

  • underfined 类型
    var num:underfined
    console.log(num)

  • null 类型

    var num:null 
var num:null | number | underfined 可以是 null number underfined

  • void
    ts 中的void 表示没有任何类型 一般用于定义方法的时候没有返回值

    js定义方法 没有返回值
function fn(){
    console.log(14232)
}

ts为了严谨 定义没有返回值的方法时用void类型 表示方法没有任何类型
function fn():void{
    console.log(11)
}
如果有返回值的话就声明返回的类型
function fn():string{
    console.log(11)
}

  • never 类型 是其他类型的子类型包括 null和undefined代表从不会出现的值 这意味着never 类型只能被never类型赋值

vue-cli3.0 的配置信息

7月28号 酷暑难耐，今天的太阳比之前格外的毒，砖也格外的烫手，我真他妈受够了天天窝在家打游戏的日子，闲来无事拿着我的mac跑汉堡王装会逼,学习一下vue-cli 3.0 的知识

安装

  • vue3.0主打无配置打包 官方称快速原型开发，通俗的说就是你不用配置任何跟打包有关的信息，只通过安装vuecli后就可以把你的vue文件打包构建，免去了繁琐的webpack配置

    安装cli
npm install -g @vue/cli
安装快速开发原型插件
npm install -g @vue/cli-service-global

安装完成以后 新建一个App.vue 文件 然后就可以通过vue serve 来进行运行了

运行项目 
vue serve  // 默认是app.vue
指定运行文件 
vue serve MyComponent.vue
打包项目 
vue build 
指定打包文件
vue build MyComponent.vue

  • 创建一个项目全局配置 

    创建一个全局项目
vue create dome

接下来vue会让你选择项目配置 根据自己的需要选择具体功能 空格选中 方向键移动 回车进入下一个配置

babel       //babel 配置
TypeScript  //支持typescript
PWA         是否支持PWA配置
Router      引入 vue—router
vuex        引入vuex
css         css 预处理器
Linter      Formatter  检查代码风格
UnitTesting Unit 测试
E2E Testing E2E  测试

  • 被保存的 preset 将会存在用户的 home 目录下一个名为 .vuerc 的 JSON 文件里。如果你想要修改被保存的 preset / 选项，可以编辑这个文件。 你可以直接使用保存的这个preset

  • 使用图形化界面

    你也可以通过 vue ui 命令以图形化界面创建和管理项目
vue ui

  • 解决浏览器兼容性

1.解决css浏览器兼容性

browserslist
在package.json

2.解决js兼容性

// babel.config.js 在babel.config.js 里面进行兼容新的配置
module.exports = {
presets: [
    ['@vue/app', {
    polyfills: [
        'es6.promise',
        'es6.symbol'
    ]
    }]
]
}

git 常用命令

  • 初始化 

    git config --global user.name '用户名'
git config --global user.email '邮箱'

  • 生成秘钥 

    ssh-keygen -t rsa

  • 链接远程仓库

    移除当前远程仓库 git remote rm origin 
链接新的远程库   git add origin  git@github.com:djqiang/gitdemo.git

  • 查看当前修改状态 

    git status

  • 分支操作

新建分支 git branch dev 
切换分支 git checkout dev 
查看分支 git branch 
删除远程分支 git push origin --delete dev 
删除本地分支 git branch -d
同步远程分支 git fetch origin dev 
新建并切换远程分支 git checkout -b dev origin/dev 
拉取远程分支 git pull origin dev 
合并分支 git merge dev  （把dev的代码合并到当前分支）
