vue-cli3.0 的配置信息

7月28号 酷暑难耐，今天的太阳比之前格外的毒，砖也格外的烫手，我真他妈受够了天天窝在家打游戏的日子，闲来无事拿着我的mac跑汉堡王装会逼,学习一下vue-cli 3.0 的知识

安装

  • vue3.0主打无配置打包 官方称快速原型开发，通俗的说就是你不用配置任何跟打包有关的信息，只通过安装vuecli后就可以把你的vue文件打包构建，免去了繁琐的webpack配置

    安装cli
npm install -g @vue/cli
安装快速开发原型插件
npm install -g @vue/cli-service-global

安装完成以后 新建一个App.vue 文件 然后就可以通过vue serve 来进行运行了

运行项目 
vue serve  // 默认是app.vue
指定运行文件 
vue serve MyComponent.vue
打包项目 
vue build 
指定打包文件
vue build MyComponent.vue

  • 创建一个项目全局配置 

    创建一个全局项目
vue create dome

接下来vue会让你选择项目配置 根据自己的需要选择具体功能 空格选中 方向键移动 回车进入下一个配置

babel       //babel 配置
TypeScript  //支持typescript
PWA         是否支持PWA配置
Router      引入 vue—router
vuex        引入vuex
css         css 预处理器
Linter      Formatter  检查代码风格
UnitTesting Unit 测试
E2E Testing E2E  测试

  • 被保存的 preset 将会存在用户的 home 目录下一个名为 .vuerc 的 JSON 文件里。如果你想要修改被保存的 preset / 选项，可以编辑这个文件。 你可以直接使用保存的这个preset

  • 使用图形化界面

    你也可以通过 vue ui 命令以图形化界面创建和管理项目
vue ui

  • 解决浏览器兼容性

1.解决css浏览器兼容性

browserslist
在package.json

2.解决js兼容性

// babel.config.js 在babel.config.js 里面进行兼容新的配置
module.exports = {
presets: [
    ['@vue/app', {
    polyfills: [
        'es6.promise',
        'es6.symbol'
    ]
    }]
]
}

<!-- 解决css兼容性 -->
修改.browserslistrc 文件里面的内容

"> 1%",
"Android >= 4.0",
"IOS >=4.0",
"last 5 Firefox versions",
"Firefox >=10",
"Chrome >10",
"Safari >= 6",
"ie >=9"

3.vue.config.js 是一个可选的配置文件
