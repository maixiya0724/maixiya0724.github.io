js自运行函数原理及用法

自执行函数是什么?自执行函数就是当它被定义出来，就会自动执行的函数，不需要再次调用。

上面就是一个典型的立即执行函数

  • 首先声明一个匿名函数function(){alert(‘我是匿名函数’)}
  • 然后在匿名函数后面接一对括号()，调用这个匿名函数

那么为什么还要用另一对括号把匿名函数包起来呢

其实是为了兼容 JS 的语法。如果我们不加另一对括号,浏览器会报语法错误。就是为了通过浏览器的检查。
想要通过浏览器的语法检查，必须加点小东西，当函数被理解成表达式时就会执行,比如下面几种

(function(){alert('我是匿名函数')} ()) // 用括号把整个表达式包起来
(function(){alert('我是匿名函数')}) () //用括号把函数包起来
!function(){alert('我是匿名函数')}() // 求反，我们不在意值是多少，只想通过语法检查。
+function(){alert('我是匿名函数')}()
-function(){alert('我是匿名函数')}()
~function(){alert('我是匿名函数')}()
void function(){alert('我是匿名函数')}()
new function(){alert('我是匿名函数')}()

立即执行函数有什么用？

只有一个作用：创建一个独立的作用域。

这个作用域里面的变量，外面访问不到（即避免「变量污染」）,一般用于封装插件的时候使用。

自执行函数有三种写法

  • 1.( function ( “ 参数 ” ) { “ 函数方法 “ ; } ) ( “ 给参数传的值 ” )

  • 2.( function ( “ 参数 “ ) { “ 函数方法 ” ; } ( “ 给参数传的值 “ ) )

  • 3.! function ( “ 参数 “ ) { “ 函数方法 “ ; } ( “ 给参数传的值 “ )　

第三种!可以换作其他运算符或者void。

自执行函数是很自私的，它的内部可以访问全局变量。但是除了自执行函数自身内部，是无法访问它的，所有的变量都是内部变量，无法在外部调用自运行函数和获取自运行函数的值.

1.(function(window){
    var a = 2;
    window.a = a
})(window)

2.(function(window){
    var a = 2;
    window.a = a
}(window))

3.!function(window){
    var a = 2;
    window.a = a
}(window)
